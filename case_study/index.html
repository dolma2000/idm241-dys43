Overview

IDM 241 is a “Micro Interaction” course where I learnt to identify and implement the elements required to design an incredible digital experience. I have explored many modern design theories and scripting languages to complete this project of creating 3 micro interactions, 2 of which I have merged to become my final complex micro interaction.

Background

In this class, we are asked to develop 3 micro interactions: alpha, beta, and final by ourselves. Each stage increases in complexity depending on the number of rules required in the micro interaction. These should be referenced from existing micro interaction and improved into a more enjoyable experience.

The purpose of this project, which is also my motivation, is essentially to explore modern design theories and languages that allows user to enjoy meaningful micro interactions.

The timeline for developing these micro interaction are varied: alpha (2 weeks), beta (3 weeks), and final (2 weeks). The alpha I made is its own micro interaction. For my beta and final, I have merged them together as one complex set of micro interaction.

This project is successful not only if my code worked, but also if my designed improved micro interaction left an impression to my users or not.

Target Audience

The interactions I have made are all to be included in my portfolio website eventually. Because of this, my target audience are professionals/potential employers who want to judge me for my skill and talent. Thinking with this mindset, I was motivated to create a very fun and animated interaction to portray my skills.

ALPHA

HAMBURGER MENU BAR

Context

The micro interaction I want to improve is based off the hamburger menu bar from my current website: My Portfolio Website

Problem: Currently, when clicking on the hamburger, a menu bar just slides in covering over half of the page. The hamburger button also does not give a direct feedback. To conclude, this interaction is very simple, predictable, and not exciting at all.

Proposed Improvement: I decided to add two rules. First is the hamburger icon, as it is a button, naturally people would want a kind of immediate direct feedback upon tapping it. So, I want to have the hamburger icon to have its own animation where it would transition to an “X” upon clicking it. Second is the menu bar, initially I wanted the bar to slide in like it does originally but only have the text come in one by one. But this changed later on as I began to explore more designs online.

I believe these changes will add a sense of joy and life.

Process

CODE / DEVELOPMENT

I started with coding the hamburger and then the menu bar. The improved hamburger icon animation was pulled from jonsuh.com which has a set of different hamburger icon animations. The one I chose was called “Spin” that transitions from a hamburger icon to an “X” by spinning. I implemented this animation’s CSS and JavaScript code as directed from the jonsuh.com website.

Now that was done, I began to work on the menu bar sliding in. However, before starting this, I came across an Adobe XD micro interaction animation that inspired me to change my initial proposed idea.

I really loved how the bars come in one by one and they each have different colors too. It’s smooth animation makes it very attractive and elegant, a style I love. I already have the words coming in one by one, but this inspired me to up my design. I have decided, I will make each word have its own colored bar and the bar will come in one by one instead!

To build this animation, I first made sure each word is in its own div and has its own “id”. That way I could stylize them to become a bar by adding “padding” and refer to their “id” to give them a specific “background color”.

All of these together are enclosed within a “ul” tag which allows them to appear as a list. The “ul” tag and each div that is enclosed by a “li” tag is overall surrounded by a “nav” tag with a class = “nav-links”. The “id” for each bar is set to transform:translateX(-100%) so it is all located out of the screen on the left side. To have the bar slide in with full view, I made a separate class “.nav-active” which is not present in the html page. I used Javascript to make the classList of the nav bar toggle between “.nav-links” and “.nav-active”. The “.nav-active” have “id” of the bars coded with different CSS style. The CSS code for the .nav-active #first #second #third all have their transform property changed to translateX(0%) so that it can slide into the screen. So if we click on the hamburger icon, the 3 bars will appear at once . What I wanted was for them to come in one by one. To achieve this, I set the 2nd bar to have a transition delay of 0.2s and the 3rd with 0.4s.

CHALLENGE

The challenge for my alpha was sliding the nav bar out of view. What happened was that the whole bar slide out together at once; I wanted them to leave one by one as well. So I just copied the transition delay from the .nav-active bar styling and pasted it into the .nav-links bar styling. However it turned out like this:

SOLUTION

I did not like it and it took me a while to figure out, but I realized that the order of the bar leaving is different from them coming in. So, to fix this, for the .nav-links #first #second #third, I set the 1st bar to have a transition delay of 0.4s and the 2nd bar with 0.4s. This time, it is the 3rd bar that has no transition delay.

BETA + FINAL

BLOB

Context

The Micro-Interaction I want to improve is a blob from the UI Kit website: UI Motion Kit

Problem: Just a normal animation with no interaction.

Proposed Improvement: It looks really beautiful but I believe it has potential for improving the experience by making it interactive. First once you hover on top of it, I want to make it so that the blob and text increase in size and the color of the blob will change too.

Second, I want add a click trigger where the blob will fill up the whole screen into a new page where the content inside will appear.

Process

HOVER INTERACTION

The animation of the blob was pulled from code pen. I copied and pasted the code to my own codepen board in my dashboard and started editing there.

First, I wanted the blob to have a bouncy animation on hover. So visually the size of the blob would increase a bit, then decrease a bit and then increase once more again. To do this, I used the animation CSS property and the keyframe rule. I called my animation name “bounce” and changed the size using transform:scale property.

To change the background color, I simply set the background color to be the vibrant blue for when the blob is on hover. The same goes for changing the color of the text from black to white.

Challenge

My main challenge was that when I added the word “Photos”, it wouldn’t move along with the blob, appearing static and awkward. Even though I did apply the same “movement” animation to this word, it still stayed still. Later I realized that this was because “movement” animation have a translateY transform property of only 8%. For the blob, the change in the Y axis can be seen, however, the word photo is only 50px in height and therefore, though the animation “movement” is being applied to it, its movement is easily missed.

Solution

After realizing the main cause of this issue, I made a new animation for the word “Photos” to move. I basically copied the “movement” animation but renamed it as “movements” and changed the translate transform property from 8% to 90%. This makes the Y axis movement of the word more visible and in sync with the blob moving.


CLICK INTERACTION

For the first interaction, I first developed the new page animation where the word “Photos” and the three images will fade in upwards smoothly. The animation code for the word to transition in was pulled from Moving Letters Resources which is a resource of “moving letters” css animation made by as Swedish design Tobiah Ahlin. As the word was smoothly transitioning in, I also applied the same javascript code for the word animation onto the images, except the images all come at once.

Challenge

The challenge when working with the click interaction was for making the blob fill up the page on click. My main issue was that the CSS styling for the blob that I copied from code pen was developed from the on .blob::before class and it is very difficult to work with a pseudo class.

What I initially did was made another pseudo class called blob:after as I was hoping I could use Javascript to toggle the classlist from .blob::before to .blob:after but that it’s technically not possible to access a pseudo class from JavaScript because it is not part of the DOM tree

Solution

What I did instead was created a new class called blob-full-screen and used JavaScript to change the class of the blob from blob::before to blob-full-screen onclick. Blob-full-screen class had the exact same code as blob::before, the difference is that blob-full-screen has a transform:scale property so that it increases in size till it fills up the whole page.

Looking back at my project, there are still several rooms for improvement. For example, in my Blob “Hover Interaction” paragraph, I have two animations name that are almost the same, “movement” and “movements”. Next time, I would make sure to use different names for animations so that when I come back to this code, I wouldn’t get confused, and also to prevent confusion from other people viewing my code. I would also try too make my onclick blob interaction to bounce a little bit back before filling up the screen – similar to the bounce animation of my blob on hover. Overall, I am still very happy to learn more and make improvements whenever I can.
